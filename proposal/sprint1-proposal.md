# Sprint 提案書: AI 聲音克隆系統 — Sprint 1「訓練基礎建設」

> **提案人**: 老闆 + PM
> **日期**: 2026-04-03
> **專案**: 聲音 Podcast 模擬器（voice-clone）
> **狀態**: 待 G0 審核

---

## 1. 目標

建立一套本地端 AI 聲音克隆系統，讓你輸入文案後，能自動以你的聲音朗讀並輸出 MP3。
本 Sprint 聚焦「基礎建設」：從 MP3 素材準備、模型訓練、到能跑出第一句克隆語音的完整鏈路。

**成功定義**: 輸入一段中文文字，系統輸出一段聽起來像你聲音的 MP3，無明顯機器感。

---

## 2. 技術選型決策（PM 建議，老闆確認）

| 項目 | 選擇 | 理由 |
|------|------|------|
| 聲音克隆引擎 | **GPT-SoVITS** | 開源免費、專為中文優化、社群活躍、支援少量素材 fine-tune |
| 自動轉錄 | **Faster-Whisper（large-v3）** | 本地跑、中文準確率高、快速 |
| 音訊處理 | **FFmpeg + pydub** | 格式轉換、切段、音量標準化 |
| 介面 | **Gradio 本地 Web UI**（Sprint 1 附帶） | 快速可用、不需部署 |
| 開發語言 | **Python 3.10+** | GPT-SoVITS 官方支援環境 |
| 硬體 | **GPU 電腦**（VRAM ≥ 8GB 建議） | 訓練與推論都需要 CUDA |

---

## 3. 系統架構（三階段流程）

```
[你的原始 MP3]
      ↓
【Stage 1: 資料準備】
  - 音訊切段（3~10 秒/段）
  - Faster-Whisper 自動轉錄（逐段文字）
  - 產出訓練資料集（音訊 + 文字對）
      ↓
【Stage 2: 模型訓練】
  - GPT-SoVITS Fine-tune（你的聲音）
  - SoVITS 模型 + GPT 模型 各一份
      ↓
【Stage 3: 推論介面】
  - 輸入文案 → 選參考音訊 → 生成 MP3
  - Gradio 本地 Web UI
  - CLI 指令（批次處理用）
```

## 8. 風險評估

| 風險 | 可能性 | 影響 | 緩解措施 |
|------|--------|------|---------|
| GPU VRAM 不足（< 6GB） | 中 | 高 | 改用 RTX 3060 12GB 以上；或用 FP16 省 VRAM |
| 轉錄錯誤率過高 | 低 | 中 | 人工校稿 + Whisper large-v3 中文準確率 > 95% |
| 訓練後聲音不像 | 中 | 高 | 增加訓練資料量；調整 epoch 數；選更接近的參考音訊 |
| GPT-SoVITS 安裝失敗 | 低 | 中 | 有官方 Docker 版本作為備案 |
| 中文韻律不自然 | 中 | 中 | 調整推論時的語速參數；更換參考音訊 |

---

## 9. 失敗模式分析

| 失敗場景 | 可能性 | 影響 | 偵測方式 | 緩解措施 |
|---------|--------|------|---------|---------|
| 訓練資料不足（< 30 分鐘有效）| 低 | 高 | 切段後統計總時長 | 補充錄音 |
| 模型過擬合（只能念訓練文字）| 中 | 高 | 測試未見過的文字 | 調低 epoch，增加資料多樣性 |
| 輸出有嚴重電音/雜訊 | 低 | 高 | 聆聽測試 | 重新選參考音訊、調整推論參數 |
| CLI 腳本相依性衝突 | 低 | 中 | 執行報錯 | 固定 requirements.txt 版本 |

## 10. 可觀測性

- **訓練日誌**: TensorBoard 記錄 loss 曲線，確認收斂

- **品質評分**: 主觀 MOS 評分表（1-5 分）

## 11. Rollback 計畫

| 項目 | 說明 |
|------|------|
| 模型回滾 | 保留每個 checkpoint，可隨時切換回舊版 |
| 資料回滾 | 原始 MP3 永遠保留在 `data/raw/`，不刪除 |
| 判斷標準 | 主觀評分 < 3 分 → 回到上一個 checkpoint 重試 |
